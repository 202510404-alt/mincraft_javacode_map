📜 개정판: 우리가 준수해야 할 4대 절대 코드 규칙
앞으로 코드를 생성할 때 인공지능과 개발자 모두가 칼같이 지켜야 할 프로토콜입니다.

🧩 1. 1파일 1심볼 극단 분리 규칙 (신설)
내용: 하나의 파이썬 파일에는 단 하나의 명확한 함수(또는 단일 책임을 지는 1개의 클래스/심볼)만 존재하도록 코드를 극단적으로 찢어서 배치합니다. 복잡한 거대 기능은 여러 파일로 분할되어 각각 독립된 함수로 격리되어야 합니다.

핵심: 파일명이 곧 함수의 이름이자 역할이 되도록 설계합니다. (예: clean_markdown.py, atomic_write.py)

이유: 외부 장부 시스템(.jjap_symbols.json)이 파일 단위로 컨텍스트를 징집할 때 토큰 낭비를 0%로 수렴시키고, LLM이 단 50줄짜리 순수 함수만 바라보게 만들어 '환각(Hallucination) 발생률 0%'에 도전하기 위함입니다.

🎛️ 2. On/Off 디버깅 로그 도배 규칙
내용: 오류 발생 시 핀포인트로 추적할 수 있도록 파이썬 표준 logging이나 디버깅 출력문을 코드 곳곳에 촘촘히 도배합니다.

핵심: DEBUG_MODE = True/False 같은 원터치 스위치를 장착하여, 평소에는 로그를 끄고 에러를 잡을 때만 켜서 볼 수 있도록 설계합니다.

📝 3. 주석의 엄격한 이원화 (검열 목록 추가 및 강화)
# HISTORY: (AI 오답노트): 과거에 발생했던 오류와 해결 방법을 자연어로 간략하게 기록합니다. 토큰(비용)을 아끼기 위해 극단적으로 압축하여 최적화합니다. Gemini가 같은 실수를 반복하지 않게 만드는 반성문입니다. (이 주석은 극단 분리된 파일 상단에 박혀 LLM에게 상시 배송됩니다.)

# INFO: (개발자 전용 설명서): 형님이 읽기 편하도록 초등학생에게 설명하듯 쉽고 친근하게 코드 기능을 설명합니다. 이 주석은 장부 가공 단계에서 자동으로 싹둑 잘려 나가는 최적화 영역이므로 토큰 걱정 없이 마음껏 친절하게 씁니다.

⚡ [형님의 초정밀 최적화 특명]: context_builder.py가 코드를 정화할 때 # INFO:뿐만 아니라 if DEBUG_MODE:로 시작하는 디버깅 로그 블록 전체를 흔적도 없이 도려내도록 검열 목록을 확장합니다. 1파일 1함수로 쪼개진 청정 소스코드에서 로그 블록까지 증발하므로, AI에게는 오직 순수 핵심 로직(Raw Pure Logic)만 단 몇십 줄 단위로 초압축 배송됩니다.

🎯 4. 초심 유지 규칙 (blueprint.txt & plan.txt 준수)
내용: 실제 개발 과정에서 세부적인 코드 모양은 유연하게 변할 수 있으나, 명세서에 담긴 "최소 토큰 최대 맥락", "Core-First (UI 분리)", "Safety-First (사용자 승인 및 완벽 복구)"라는 본질적인 의도와 철학은 무조건 코드에 100% 반영합니다.
